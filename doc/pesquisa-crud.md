<h1 align="center">CRUD</h1>

<H3 align="center">Create (Criação) </H3>

<p align="center"> Esta operação é responsável por inserir novos registos no sistema.
Através do método POST, a aplicação recebe os dados enviados pelo utilizador,
valida-os e armazena-os na base de dados, permitindo que novas informações
(como utilizadores ou produtos)
passem a existir no ecossistema do projeto. </p>

<H3 align="center"> Read (Leitura) </H3>

<p align="center"> A leitura permite a consulta e visualização dos dados armazenados.
Utilizando o método GET, o sistema recupera as informações da base de dados
para as exibir na interface, seja através de uma listagem geral de todos os
itens ou da procura detalhada de um registo específico via ID. Update (Atualização) </p>

<H3 align="center"> Update (Atualização) </H3>

<p align="center"> A operação de atualização serve para modificar dados já existentes sem criar
um novo registo. Através dos métodos PUT ou PATCH, o sistema localiza o 
item original e substitui as suas informações pelos novos valores fornecidos,
garantindo que o conteúdo se mantenha atualizado e relevante. </p>

<H3 align="center">Delete (Remoção)  </H3> 

<p align="center"> Esta funcionalidade encarrega-se de eliminar definitivamente um registo do sistema.
Ao acionar o método DELETE, a aplicação remove a informação correspondente 
da base de dados, garantindo que itens desnecessários sejam apagados e não 
apareçam mais nas futuras consultas de leitura. </p>
